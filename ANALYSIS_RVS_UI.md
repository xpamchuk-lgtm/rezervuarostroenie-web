# Анализ локального калькулятора и переработка web-версии

## Что есть в локальном калькуляторе (`calculator.py`)

### Структура вкладок РВС
1. `Общие сведения`
2. `Район установки`
3. `Стенка`
4. `Днище`
5. `Крыша`
6. `Металлоконструкции`

### Что включает каждая вкладка

**Общие сведения**
- диаметр резервуара (типовой / другое)
- высота стенки (типовая / другое)
- заполнение (% или уровень)
- полный объем
- полезный объем
- уровень налива
- среда (продукт)
- плотность

**Район установки**
- регион / субъект РФ
- город / населенный пункт
- ветровой район
- нормативное давление ветра `w0`
- снеговой район
- нормативный вес снегового покрова `Sg`
- температура наиболее холодной пятидневки `t5`
- абсолютный минимум `t_min_abs`
- сейсмичность
- степень сейсмоопасности

**Стенка**
- метод изготовления стенки
- таблица поясов снизу вверх
- высота пояса
- минусовой допуск
- марка стали
- `t_min`
- желаемая толщина
- коррозия
- итоговая толщина
- избыточное давление в газовом пространстве `p`
- испытательное давление `pисп`

**Днище**
- исполнение с окрайкой / без окрайки
- предупреждение по применимости днища без окрайки
- минимальная ширина окрайки `L0`
- строка `Днище`
- строка `Окрайка`
- для каждой строки: сталь, минусовой допуск, `t_min`, желаемая толщина, коррозия, итоговая толщина

**Крыша**
- тип кровли: коническая / коническая каркасная / каркасно-щитовая
- уклон `α`
- настил кровли
- каркас (для каркасных типов)
- уторный уголок
- снеговая нагрузка
- масса снега
- расчетная масса кровли
- пользовательская масса кровли
- используемая масса кровли

**Металлоконструкции**
- площадка обслуживания
- лестница
- площадка обслуживания пеногенератора
- молниеприемник
- кольцевая труба орошения

## Что было не так в web-версии до переработки
- слева были разделы, которых нет в локальном калькуляторе (`Геометрия`, `Площадки`, `Нагрузки`, `Проверки` как отдельные разделы)
- `Общие сведения` и `Район установки` были сломаны на разные несоответствующие блоки
- по центру форма не менялась при клике по левому меню
- на страницу РВС было вынесено только несколько полей, тогда как локальный калькулятор содержит десятки полей и зависимостей
- режим задания уровня работал некорректно из-за несовпадения `fill_mode` между frontend и backend

## Что переработано в этом обновлении
- левая навигация приведена к реальной структуре локального калькулятора:
  - Основные данные
  - Стенка
  - Днище
  - Крыша
  - Металлоконструкции
  - Итоги и проверки
- центральная часть теперь **действительно переключает формы** при выборе раздела
- `Основные данные` объединяют:
  - геометрию
  - заполнение
  - среду
  - район установки
- `Стенка` содержит таблицу поясов с автоматической генерацией по высоте
- `Днище` содержит исполнение с окрайкой / без окрайки и расчетные строки
- `Крыша` содержит реальные типы кровли из локальной версии и связанные поля
- `Металлоконструкции` содержит все основные блоки из локального калькулятора
- добавлен backend endpoint с регионами/городами
- добавлен backend endpoint подстановки нормативов по региону/городу
- исправлено несовпадение `fill_mode=level`
