# Аудит и апгрейд проекта `rezervuarostroenie_web`

Дата: 2026-03-13

## Что было выявлено

1. **Сайт и отчет жили на разных расчетных ядрах**.
   - Экранные расчеты шли через `legacy_engine.py`.
   - DOCX-отчеты собирались на другой логике.
   - Из-за этого сайт мог показывать одно, а отчет — другое.

2. **Расчет стенки был слишком упрощен**.
   - Толщина оценивалась по простой окружной формуле без полноценного учета минимальной конструктивной толщины по диаметру, минусового допуска и припуска на коррозию.
   - Не было разделения требуемой толщины для эксплуатации и гидроиспытания.

3. **Отсутствовал контроль полноты обязательных исходных данных**.
   - Не контролировались ключевые поля из п. 5.1.8 ГОСТ 31385-2023: срок службы, оборачиваемость, вакуум, параметры среды, привязка к площадке, параметры коррозии, тип резервуара и др.

4. **Нормативная привязка по площадке была частично сломана**.
   - Снежные данные читались по неправильному имени поля CSV.
   - Поиск сейсмичности был неустойчив из-за кривых строк в исходном файле.
   - Не использовался расширенный набор норм по geofill-таблице как резервный источник.

5. **Сайт визуально выглядел как полноценный нормативный расчет, но фактически покрывал лишь часть проверок**.
   - Это опасно для доверия к отчету и для реального проектирования.

## Что изменено

### Backend

#### `backend/app/engine/strength_engine.py`
Переписано расчетное ядро инженерной проверки:
- добавлен перечень актуальной нормативной базы;
- добавлено автоматическое определение класса резервуара по объему;
- реализован расчет поясов стенки с учетом:
  - минимальной конструктивной толщины по ГОСТ 31385-2023;
  - минусового допуска;
  - припуска на коррозию;
  - отдельной проверки эксплуатации и гидроиспытания;
- добавлен контроль полноты исходных данных;
- добавлены экспертные флаги и замечания;
- добавлен явный список того, **что проверено**, и **что должно считаться отдельно проектом**.

#### `backend/app/reporting.py`
Переработаны DOCX-отчеты:
- отчет оформлен как **самостоятельное экспертное заключение**, а не стилизация под ПАССАТ;
- добавлены:
  - нормативная база;
  - границы применимости;
  - полнота исходных данных;
  - таблица замечаний экспертизы;
  - раздел с выводами и рекомендациями;
- структура документа специально сделана отличной от типовой пояснительной записки ПАССАТ.

#### `backend/app/api/routes.py`
- добавлен endpoint `POST /api/rvs/review` для единой инженерной проверки модели;
- исправлен подбор норм по площадке;
- добавлен резервный lookup по geofill-CSV;
- исправлено чтение снеговых и сейсмических данных.

### Frontend

#### `frontend/src/pages/RvsCalc.tsx`
- теперь кнопка расчета вызывает backend-проверку, а не только локально собирает геометрию;
- добавлены поля проектной постановки:
  - тип резервуара;
  - класс резервуара;
  - срок службы;
  - циклы/год;
  - вакуум;
  - коэффициент трения;
  - допускаемое давление на основание;
  - тип местности;
  - ГО/УЛФ/инертирование;
  - температуры продукта и окружающей среды;
- на экране результатов теперь выводятся:
  - итог инженерной проверки;
  - полнота исходных данных;
  - перечень автоматических проверок;
  - замечания экспертизы;
  - граница между укрупненной оценкой и полноценным проектным расчетом.

## Что в проекте по-прежнему должно считаться отдельно

Даже после апгрейда сайт **не заменяет** полноценный проект КМ/КМД и не должен выдавать себя за него без оговорок. Отдельно остаются обязательными:
- полный расчет сейсмостойкости резервуара;
- расчет устойчивости стенки от ветра и вакуума;
- расчет элементов крыши и каркаса;
- расчет анкеров и узла стенка-днище;
- геотехнический расчет основания и фундамента по ИГИ;
- подбор оборудования безопасной эксплуатации;
- комплект обязательной документации по приложению Е ГОСТ 31385-2023.

## Важный содержательный вывод по вашему текущему отчету

Для модели с продуктом **«Бензин»** исходная версия отчета выглядела благополучной, но после профессиональной перепроверки выявляется, что:
- отсутствует подтверждение допустимого типа резервуара для данной среды;
- не закрыт набор обязательных исходных данных по ГОСТ;
- стенка 5 мм в нижнем поясе на вашей модели уже не проходит как «суперпрофессиональное» решение при учете требуемой номинальной толщины, если вести проверку строже и отдельно учитывать эксплуатацию/гидроиспытание.

## Рекомендуемый следующий шаг

Использовать обновленный сайт как **честный инженерный pre-check / design-review tool**, а не как замену полноценной проектной записки. Тогда он становится действительно сильным продуктом: быстро выявляет проблемные места, собирает корректную расчетную базу и формирует профессиональные документы без копирования ПАССАТ.
