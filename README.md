# rezervuarostroenie.ru — платформа калькуляторов (v0.1 scaffold)

Это стартовый проект, который:
- сохраняет ваш расчётный алгоритм (rvs_calc/rgs_calc) в backend (FastAPI);
- даёт красивый веб-интерфейс (React) в стиле согласованного макета;
- готов к расширению до полного покрытия вашего Tkinter-калькулятора (1:1 по расчётам и проверкам).

## Что уже сделано
1) **Расчётное ядро вынесено** в `backend/app/engine/legacy_engine.py`:
   - из вашего `legacy/calculator.py` автоматически извлечены чистые функции и константы;
   - UI (Tkinter) не импортируется и не требуется.

2) **API**
   - `GET /api/health`
   - `POST /api/calc/rvs`
   - `POST /api/calc/rgs`

3) **Frontend**
   - Главная (каталог калькуляторов): `/`
   - РВС-калькулятор: `/calc/rvs` (ввод ключевых параметров + блоки сводки/итогов/проверок)
   - заложены рекламные слоты (пока пустые).

## Быстрый запуск на вашей VM (Yandex Cloud)
Требования: Docker + compose plugin.

```bash
git clone <ваш-репозиторий> rezervuarostroenie
cd rezervuarostroenie
docker compose up -d --build
```

Открыть:
- `http://<IP>/` — главная
- `http://<IP>/calc/rvs` — калькулятор
- `http://<IP>/api/health` — API

## Как довести до 1:1 с текущим Tkinter
1) В вашем `legacy/calculator.py` много UI-кода (класс `App`).
   В нём расчёты/проверки уже частично вынесены в чистые функции `rvs_calc`/`rgs_calc`.
2) Дальше работаем по секциям:
   - выделяем “пэйлоад” (все поля формы) в pydantic-модели
   - расширяем ответ API (`summary/details/checks`)
   - фронтенд получает 1:1 те же поля и те же тексты предупреждений.

## Важно
Сейчас в веб-форме выведены ключевые поля для RVS. Остальные разделы (лестницы/площадки/спецификация/PDF) добавляются следующими итерациями, но каркас под них уже заложен.
